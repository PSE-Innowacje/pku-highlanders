# Iteration 01 — Administrator manages contractor types

## Goal
We want to have functionality, where user in role Administrator can manage contrator types (see PRD point 4).

## Requirements
- We have initial list of contractor types (see PRD point 4), which can be extended.
- application will have menu on left hand side, where user can navigate between viewes.
- user in role Administrator will have new view "Contractor types" ("Typy kontrahentów") in webapp where he can add new contractor types. Initial contractor types cannot be removed or edited. Only added contractor types can be edited or removed.
- user in role Kontrahent does not have access to this view
- application should by default use Polish language

## Acceptance criteria
- user in role Administrator after logging in can go the "Contractor types" view, review and edit.
